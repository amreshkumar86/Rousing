Overview

ponut is a fully featured premium admin template built on top of awesome Cascader UI Framework, modern web technology AngularJS 1.X, HTML5, CSS3 and jQuery. It has many ready to use hand crafted components. The template is fully responsive and easy to customize. The code is super easy to understand and gives power to any developer to turn this template into real web application.

Automation with Grunt

Intro

Grunt is a JavaScript task runner (Automation), save yourself from repetitive tasks. It's probably a bit complex to you if it's your first time using Grunt. However, you'll fail in love with it once you get up and running, it saves you from repetitive tasks. Projects like jQuery, Bootstrap and many more use it as their automation tool. By the way, be sure to check out this Getting Started guide, it will help you get up and running.

Prerequisites

Node.js
To be able to use ponut, you will need Node.js installed to your computer. I won't get into too much detail about Node.js as it's out of the scope of this documentation. Also you won't need to actually use Node.js, it's only required for the development process.

Grunt
Grunt is required for ponut to work properly. Although there are ways to develop with ponut without using Grunt or Node.js, I strongly recommend you to use them in order to take advantage of the true power of the ponut.

Explanation

Gruntfile.js : is the configure file of Grunt. It is used to configure or define tasks and load Grunt plugins.
package.json : is the JSON file of NPM. This file is used by npm to store metadata for projects published as npm modules. Inside package.json file, you'll see all the plugins that help to automate the tasks. And all the plugins are configured in Gruntfile.js and Grunt folder.

Installation

After installing the prerequisites, go to the package root folder and run this command. 

npm install

Grunt Default Task

Use this task for intial local development, simply it compiles pug, stylus, run local server and watch files for any changes to compile files again and refresh your browser for instant view of the results. In this task the produced code will not be formatted, means no extra spaces will be removed, erros of indentation will remain as is but that is OK you just need it for local testing. 

grunt

Grunt Build Task

Use this task for before delivering for a client, simply it does the same as default task + formatting the code. 

grunt build

Grunt Dist Task

Use this task before publishing online for code optimizing means concatinating files, minifing CSS and JS and so on. 

grunt dist

New Page

To create a new page, you can use blank directory in scenes folder which provides basic page layout which you can extend and modify further.

CSS / Stylus

This item uses Stylus and PostCSS as base Preprocessors beside Cascader UI framework. All items parts are decomposed for small parts for modularity. The item comes with a lot of CSS Class utilities for rapid prototyping just take some time to get familir with them. you can view full list of vailble class here

The entry point to all stylus files are app.[Theme Color].styl file which imports all the necessary files for the app. so obiuosly the result is only a single CSS file for the whole app.

when you are using grunt dist task for publishing your app online, all unused CSS inside the produced app.css files will be removed.

Theming

To choose a different theme you need to load two files app.[THEME COLOR].css and app.[THEME COLOR].js and that is it.

Fonts

This template uses Source Sans Pro font as main font, Roboto Slab as serif font & Fira Mono as mono font. ponut also uses Material Icons as icon font.

Javascript

This item rely on Angular, jQuery, jQuery UI libs beside other 3rd party plugins to achieve its functionality. and here is explanation of custom written files:

First, note that all scripts are decomposed to small ones and each specific has it own functionality, for each component or page you will find separate script files for controller, directives, modules and routing

directives
Found in custom scripts folder and it has simple usage which is handle jquery plugins like flot charts

factories
factory.isMobile.js used to check for mobile devices.
factory.store.js used to store data between diffrerent isolated scopes.

filters
filter.highlight.js used to highlight searched keywords.
filter.relativeTime.js used to display time in ago format.
filter.search.js advanced search filter for searching specific properties in array of objects.
